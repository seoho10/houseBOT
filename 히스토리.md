- 또는 네이버가 본인 PC IP도 차단 — 다음날까지 기다리면 풀림

### 회사 PC 셋업 (월요일 출근 후)
1. `git clone https://github.com/seoho10/houseBOT.git`
2. `cd houseBOT`
3. `pip install -e ".[dev]"` (Python 3.11+ 필요)
4. `.env` 파일 만들기 (개인 PC `.env`에서 그대로 복사)
5. Google SA JSON 파일도 안전한 위치에 (예: `C:\Users\<user>\google-sa.json`)
6. **2분 시차 두고** 작업 스케줄러 등록 (개인 PC와 충돌 방지):
   ```powershell
   powershell -ExecutionPolicy Bypass -File scripts\install-tasks.ps1 -DailyAt "08:32" -CheckOffsetMinutes 2
   ```
7. Smoke test: `Start-ScheduledTask -TaskName "houseBOT-Daily"`

## 7. 트러블슈팅

### "Naver fetch failed for complex ... 429 Too Many Requests"
- 잠시 IP 차단됨. 1~12시간 기다리면 풀림.
- 자주 발생하면 `scripts/install-tasks.ps1` 트리거 간격 늘리기.

### "Naver fetch failed for complex ... read operation timed out"
- 네이버가 응답 안 줌 (IP 의심 가능성).
- 차단 풀릴 때까지 대기.

### "Daily already ran within last 6h on another host - skipping"
- 정상 동작. 다른 PC가 이미 실행했음.
- 강제로 다시 돌리려면: Google Sheets `run_log` 탭의 마지막 daily SUCCESS row를 수동 삭제.

### 작업 스케줄러가 실행 안 됨
- PC 켜져있는지 확인
- `python` 명령어가 PATH에 있는지 확인 (`python --version` 시도)
- `.env` 파일이 `C:\Users\user\Desktop\project\houseBOT\.env`에 있는지 확인
- 작업 스케줄러 GUI → 해당 작업 → "마지막 결과" 확인
